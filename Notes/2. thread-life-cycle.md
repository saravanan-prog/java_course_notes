## 🔹 Thread Life Cycle

A thread goes through different states:
```
New             – Thread object created
Runnable        – Ready to run
Running         – Currently executing
Blocked/Waiting – Waiting for resources
Terminated      – Finished execution
```
#### 🔹 Important Methods
```
    start()     → starts a new thread
    run()       → contains thread logic
    sleep(ms)   → pauses execution
    join()      → waits for another thread to finish
    isAlive()   → checks if thread is running
```

## 🔹 start() → starts a new thread

- When you call start(), 
- Java creates a new separate thread and then internally calls run().
```
Thread t = new Thread(() -> {
    System.out.println("Running in new thread");
});
t.start();
```
👉 Key point:
```
- start()   = new thread begins execution
- You can call start() only once per thread (else → exception)
```

## 🔹 run() → contains thread logic

This is just a normal method that holds the code to execute.

```
t.run();  // NOT a new thread!
```
#### 👉 Important:

- Calling run() directly does not create a new thread
- It runs in the current thread (main thread)

#### ✔ Think of it like:
```
start() → “Do this in parallel”
run() → “Just execute normally”
```


## 🔹 sleep(ms) → pauses execution

Pauses the current thread for a given time (milliseconds).
```
try {
    Thread.sleep(1000); // 1 second
} catch (InterruptedException e) {
    e.printStackTrace();
}
```
#### 👉 Key points:

- Does not release locks (important in synchronization)
- Always wrapped in try-catch
- Used for delays, timing, simulations

## 🔹 join() → waits for another thread to finish

Makes one thread wait until another thread completes.
```
Thread t = new Thread(() -> {
    System.out.println("Child thread running");
});

t.start();
t.join();  // main thread waits here

System.out.println("Main thread continues");

👉 Output order will always be:

Child thread running
Main thread continues
```
#### ✔ Useful when:

- You need results before continuing
- Coordinating multiple threads


## isAlive() → checks if thread is running

Returns true if the thread is still executing.
```
Thread t = new Thread(() -> {
    try { Thread.sleep(2000); } catch(Exception e){}
});

t.start();
System.out.println(t.isAlive()); // true (while running)
```
#### 👉 Returns:
```
true → thread is running
false → thread finished or not started
```
