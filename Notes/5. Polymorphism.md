# Java Polymorphism


Java Polymorphism is another key concept in Object-Oriented Programming (OOP).
The word polymorphism means “many forms” — the same method or object behaves differently depending on the situation.

# 🔹 Types of Polymorphism in Java

### 1. Compile-Time Polymorphism (Method Overloading)

- Same method name, but different parameters.

```
class Calculator {
    int add(int a, int b) {
        return a + b;
    }

    double add(double a, double b) {
        return a + b;
    }

    int add(int a, int b, int c) {
        return a + b + c;
    }
}

👉 Here, add() behaves differently based on input → polymorphism

```



### 2. Runtime Polymorphism (Method Overriding)

- Method behavior is decided at runtime using inheritance.

```
class Animal {
    void sound() {
        System.out.println("Animal makes sound");
    }
}

class Dog extends Animal {
    void sound() {
        System.out.println("Dog barks");
    }
}

class Cat extends Animal {
    void sound() {
        System.out.println("Cat meows");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal obj;

        obj = new Dog();
        obj.sound(); // Dog barks

        obj = new Cat();
        obj.sound(); // Cat meows
    }
}
👉 Same method sound() → different behavior → runtime polymorphism

```


### 🔹 Key Concept: Method Overloading Rules
- Same method name
- Different parameter list (type, number, or order)
- Return type alone is NOT enough

### 🔹 Key Concept: Method Overriding Rules
- Same method name
- Same parameters
- Must use inheritance
- Runtime decides which method to call

### 🔹 Important Keywords

 - @Override Helps avoid mistakes

```
@Override
void sound() {
    System.out.println("Dog barks");
}
```


# 🔹 Real-Life Example

Think of a Payment system:

- pay() method
- Can behave differently:
- Credit card → process card
- UPI → scan & pay
- Net banking → login & pay

Same method → different behaviors ✔️


# 🔹 Difference Between Overloading vs Overriding

| Feature     | Overloading  | Overriding   |
| ----------- | ------------ | ------------ |
| Time        | Compile-time | Runtime      |
| Inheritance | Not required | Required     |
| Parameters  | Must differ  | Must be same |
| Method name | Same         | Same         |
