# Exception?
- Exception = Error that happens while program is running
- Why Exceptions Occur:

```
    Wrong input
    Invalid operations
    Missing files
    Logical mistakes
```

```
    int x = 10 / 0; // ❌ Exception
```


# Exception Handling :
  
   - It’s a way to prevent your program from crashing when an error happens.

   - Without handling:

   ```
    int x = 10 / 0;        // ❌ program crashes

   ```

   - With handling:

   ```
    try {
        int x = 10 / 0;
    } catch (Exception e) {
        System.out.println("Error handled");
    }
   ```
 ### Why we use it?
- Prevent program crash
- Show user-friendly message
- Continue program execution



##  Types of Exceptions (Very Important)

### ✅ Checked Exception (Must handle)

- File not found
- Database error

```
    FileReader f = new FileReader("a.txt"); // must handle
```

### ❌ Unchecked Exception (Runtime)

- Divide by zero
- Null pointer

```
int x = 10 / 0;

```

### Real Example (Simple Program)

```
   public class Test {
    public static void main(String[] args) {
        try {
            int[] arr = {1, 2, 3};
            System.out.println(arr[5]); // error
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("Index error handled");
        } finally {
            System.out.println("Program finished");
        }
    }
} 

```

### Common Exceptions

| Exception                        | Meaning           |
| -------------------------------- | ----------------- |
| `ArithmeticException`            | Divide by zero    |
| `NullPointerException`           | Using null object |
| `ArrayIndexOutOfBoundsException` | Wrong index       |
| `FileNotFoundException`          | File missing      |
