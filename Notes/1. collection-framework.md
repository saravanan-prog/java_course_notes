# Collections Framework

In Java, the Collections Framework is a set of interfaces and classes used to store and manipulate groups of objects.
```
                Iterable
                    |
                Collection
          _________|_________
         |                   |
       List                Set
         |                   |
   ______|______       ______|______
  |             |     |             |
ArrayList   LinkedList  HashSet   TreeSet




                Map  (separate, not part of Collection)
           _______|_______
          |               |
       HashMap         TreeMap



```

# Main Interfaces
```
    List    → Ordered collection, allows duplicates
    Set     → No duplicates, typically unordered (except some implementations)
    Map     → Key-value pairs (keys are unique, values can repeat)

```
Common Implementations
# List implementations
### ArrayList
-    Backed by a dynamic array
-    Fast for reading (random access)
-    Slower for insertions/deletions in the middle

```
    [A][B][C][D]

    Insert X →

    [A][X][B][C][D]
        ↑ shifting (slow ❌)


```
### LinkedList
- Doubly linked list
- Faster insert/delete operations
- Slower access by index
- Set implementations

```

    [A] → [B] → [C]

    Insert X →

    [A] → [X] → [B] → [C]


```
### HashSet
- No duplicates
- Unordered
- Backed by a hash table (fast operations)
```
    Add: A, B, C, A

    HashSet:
    [A] [B] [C]

```
### TreeSet
- No duplicates
- Sorted (natural order or comparator)
- Uses a tree structure (Red-Black Tree)

```
    Add: 5, 2, 8, 1

    TreeSet:
    [1] [2] [5] [8]

```



### Map implementations

- HashMap
- Key-value pairs
- Unordered
- Allows one null key

```
    Key → Value

    [1 → A]
    [2 → B]
    [3 → C]

    Stored internally (hash buckets):

    Bucket0:  -
    Bucket1:  [1 → A]
    Bucket2:  [2 → B]
    Bucket3:  [3 → C]

```
### TreeMap
- Sorted by keys
- No null keys
- Uses a tree structure
