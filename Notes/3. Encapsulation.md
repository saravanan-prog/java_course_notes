# 🧠 What is Encapsulation?

- Encapsulation = Wrapping data (variables) + methods into a single unit (class)
- 👉 And restricting direct access to data

### 🔐 Why Encapsulation?

- Protect data 🔒
- Control access
- Improve security
- Make code maintainable

### 🔥 How to achieve Encapsulation?

- 1️⃣ Make variables private
- 2️⃣ Provide getter & setter methods

## 🚗 Example

```
class Student {
    private String name;   // private variable

    // setter method
    public void setName(String name) {
        this.name = name;
    }

    // getter method
    public String getName() {
        return name;
    }
}
🧪 Usage
public class Main {
    public static void main(String[] args) {
        Student s = new Student();

        s.setName("Saravanan");
        System.out.println(s.getName());
    }
}
📊 Output
Saravanan

```

## ❌ Without Encapsulation (Bad Practice)
```
class Student {
    public String name;  // directly accessible ❌
}
👉 Anyone can modify data → no control
```



### ✅ With Encapsulation

```
    private data + public methods

👉 You control:

    what to set
    what to get

```

### 🧠 Real-Life Analogy

#### 👉 ATM Machine 💳

- You cannot access bank balance directly
- You use methods:
- checkBalance()
- withdraw()

👉 That’s encapsulation

### 🔥 Advanced Example (Validation)

```
class Account {
    private double balance;

    public void setBalance(double balance) {
        if(balance > 0) {
            this.balance = balance;
        } else {
            System.out.println("Invalid amount");
        }
    }

    public double getBalance() {
        return balance;
    }
}

👉 You can control wrong data

```

## ⚡ Key Points

- Data hiding 🔒
- Use private variables
- Use getters/setters
- Improves security
